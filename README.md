# 🎓 院校雷达 — 考研情报自动监控系统

> 全自动监控目标院校官网动态，AI 深度解析通知内容，精准推送到你的手机/飞书/邮箱。
> 让每一条招生通知、复试安排、调剂信息都不会错过。



---

## ✨ 核心功能

| 功能 | 说明 |
| --- | --- |
| 🌐 **定向网页监控** | 只抓取你配置的院校研究生院/学院官网通知页，不会爬取任何无关页面 |
| 🤖 **AI 深度解析** | 调用大模型（支持 Gemini / DeepSeek / 通义千问 / 智谱 / Kimi / OpenAI）逐篇分析通知内容，自动打分、分类、提取关键信息 |
| 🎯 **专业精准过滤** | 自动匹配你设定的目标专业（如"0454 应用心理"），过滤掉无关专业和博士招生通知，只推送硕士相关内容 |
| ⏰ **14 天时效控制** | 自动跳过超过 14 天的旧通知，确保你收到的都是最新鲜的情报 |
| 📬 **多渠道推送** | 支持飞书机器人、邮件通知等多种推送方式 |
| 📊 **Web 管理面板** | 自带可视化后台，一键配置院校/专业/通知渠道/AI模型，实时查看扫描状态 |

---

## 🖥️ 系统要求

- **Python** 3.8 及以上版本
- **操作系统** Windows / macOS / Linux
- **网络** 能正常访问目标院校官网（如需代理请在面板中配置）

## 🚀 快速开始

### 第一步：下载项目

```bash
git clone https://github.com/your-username/university-radar.git
cd university-radar
```

### 第二步：安装依赖

> 推荐使用清华镜像源加速

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 第三步：安装浏览器内核

部分院校官网需要渲染 JavaScript，系统内置了无头浏览器兜底方案：

```bash
python -m playwright install chromium
```

### 第四步：启动服务

**方式 A：命令行启动**
```bash
python app.py
```

**方式 B：双击一键启动（推荐）**
直接双击项目根目录下的 **`一键启动.bat`** (Windows) 或 **`一键启动.command`** (macOS)。
*现在它们会由 Python 引导启动，能有效避开 Windows 11 的拦截。*

**方式 C：手动命令启动**
在终端或 PowerShell 中执行：
```bash
python start.py
```
1. 首次运行需赋予权限：打开终端，输入 `chmod +x ` (后面有个空格)，然后将 `一键启动.command` 拖入终端按回车。
2. 之后直接双击 `一键启动.command` 即可运行。

> 💡 **首次运行提示**：如果是从 Release 下载的压缩包，项目内不含 `config.yaml`。启动脚本会自动运行，但你也可以手动参考 `config.example.yaml` 进行配置。建议直接启动后通过 Web 面板配置，系统会自动生成配置文件。

## ⚙️ 配置说明

所有配置均可通过 Web 面板完成，无需手动编辑文件。启动后打开 `http://127.0.0.1:5000/terminal`，点击"设置"进行配置。

### 1. AI 引擎配置

| 配置项 | 说明 | 示例 |
| --- | --- | --- |
| AI 提供商 | 选择你使用的大模型厂商 | `Google (Gemini)` |
| API Key | 你的大模型 API 密钥 | `AIzaSy...` |
| Base URL | API 端点地址（选填，留空用默认） | `https://generativelanguage.googleapis.com/v1beta/openai/` |
| 模型 | 使用的模型名称（选填，留空用默认） | `gemini-2.0-flash` |

**默认模型对照表：**

| 提供商 | 默认 Base URL | 默认模型 |
| --- | --- | --- |
| Google (Gemini) | `https://generativelanguage.googleapis.com/v1beta/openai/` | `gemini-1.5-flash` |
| DeepSeek | `https://api.deepseek.com` | `deepseek-chat` |
| 通义千问 (阿里) | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` |
| 智谱 AI | `https://open.bigmodel.cn/api/paas/v4/` | `glm-4-flash` |
| Kimi (月之暗面) | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` |
| OpenAI | `https://api.openai.com/v1` | `gpt-3.5-turbo` |

> 💡 **提示**：如果你使用的是中转站或 OneAPI 等聚合平台，只需填写对应的 Base URL 和 API Key 即可。系统全量使用 OpenAI 兼容协议，可以对接任何支持该协议的服务。

### 2. 院校与专业配置

1. 在面板中选择你要监控的**大学**（支持按省份筛选）
2. 选择对应的**目标专业**（如 `0454 应用心理`）
3. 填写该大学的两个核心监控网址：
   - **研究生院通知页** — 如 `https://yz.xxx.edu.cn/tzgg.htm`
   - **学院/系通知页** — 如 `https://psych.xxx.edu.cn/notice.htm`

> ⚠️ **关键提示**：
> - 请务必填写准确的通知公告列表页 URL，而非学校首页
> - 系统只从你填写的这些页面入手抓取，不会爬取任何其他网站
> - 可同时监控多所大学，每所大学配置独立的专业和 URL

### 3. 通知渠道配置

在面板底部"通知渠道"区域添加你想要接收推送的平台。

#### 飞书机器人（推荐）
1. 在飞书群中添加"自定义机器人"
2. 复制 Webhook 地址
3. 在面板中选择渠道 `feishu`，粘贴 Webhook 地址到 Token 栏

#### 邮件通知
在面板中选择渠道 `email`，配置 SMTP 服务器、发件人、收件人等信息。

### 4. 扫描间隔

默认每 **15 分钟** 执行一次全量扫描。可在面板中修改，建议设置范围 5~60 分钟。

### 5. 推送阈值

`config.yaml` 中的 `relevance_threshold` 控制推送门槛（默认 60）。
AI 会对每篇通知打出 0~100 的相关度分数：
- **90+** → 高度相关的硕士招生/复试/调剂/录取信息
- **60~89** → 一般性考研通知
- **< 60** → 无关专业/博士招生/杂项通知（自动拦截，不推送）

## 🔄 工作流程

```
┌─────────────┐     ┌──────────────┐     ┌────────────────┐     ┌────────────┐
│  APScheduler │────>│  scanner.py  │────>│ ai_analyzer.py │────>│ pusher.py  │
│  定时触发     │     │  爬取指定URL  │     │  AI 打分+分类   │     │  多渠道推送  │
└─────────────┘     └──────────────┘     └────────────────┘     └────────────┘
                           │                      │                     │
                           ▼                      ▼                     ▼
                    ┌──────────────┐       ┌──────────────┐     ┌──────────────┐
                    │ 14 天日期过滤 │       │ 专业匹配拦截   │     │ 60 分阈值拦截 │
                    │ 跳过旧通知    │       │ 过滤博士/无关  │     │ 低分不推送    │
                    └──────────────┘       └──────────────┘     └──────────────┘
```

1. **定时调度**：`scheduler.py` 按设定间隔触发全量扫描
2. **精准爬取**：`scanner.py` 仅访问用户配置的 URL，提取通知列表中近 14 天的链接
3. **AI 深度分析**：`ai_analyzer.py` 将通知正文+用户目标专业发送给大模型，获取分类标签、相关度评分、干货摘要
4. **三重过滤**：日期过滤（14天）→ AI 专业匹配过滤 → 阈值分数过滤
5. **多渠道推送**：通过评分的新通知被立即推送至用户配置的通知渠道

## ❓ 常见问题

### Q: 为什么没有收到推送？
**A:** 请按顺序排查：
1. 确认已在面板中正确配置了至少一个通知渠道
2. 确认 AI API Key 有效（面板中有"测试连接"按钮）
3. 检查 `logs/radar.log` 中是否有报错信息
4. 确认推送阈值设置合理（默认 60 分），有些通知可能因为和你的专业无关被 AI 打了低分

### Q: 能监控多所大学吗？
**A:** 可以。在面板中添加多所大学即可，每所大学独立配置专业和监控 URL。系统会并发扫描所有配置的大学。

### Q: 需要代理吗？
**A:** 如果你需要通过代理访问 AI API（如 Google Gemini），可在面板中配置代理地址（如 `http://127.0.0.1:7890`）。访问国内院校官网通常不需要代理。

### Q: 支持哪些 AI 模型？
**A:** 任何支持 OpenAI 兼容协议（`/chat/completions`）的模型都可以使用。只需填写正确的 Base URL、API Key 和模型名称。

### Q: 如何更新前端页面？
**A:** 前端源码位于 `frontend/src/` 目录，修改后执行：
```bash
cd frontend
npm install
npm run build
```
构建产物会自动输出到 `frontend/dist/`，Flask 直接读取无需额外配置。

### Q: 为什么在别的电脑上双击 `一键启动.bat` 时，屏幕出现蓝色弹窗写着“Windows 已保护你的电脑”或带有禁止符号？
**A:** 这是因为文件带有“互联网下载标记 (Mark of the Web)”。Windows 拦截网 (SmartScreen) 或智能应用控制会自动拦截从互联网下载的脚本文件执行。

**解决办法（任选其一）：**
1. **彻底解封压缩包（最推荐）**：在你刚下载或接收到 `.zip` 压缩包时，**先不要解压**。右键点击那个压缩包，选择 **“属性”**。在属性窗口的最下方，如果看到一个 **“解除锁定 (Unblock)”** 的复选框，**打上勾**。点击“确定”后再解压。这样解压出来的所有文件就都安全了，可以直接运行。
2. **解封单个文件**：如果已经解压了，右键点击 `一键启动.bat` 文件，选择 **“属性”** -> 勾选最下方的 **“解除锁定 (Unblock)”** -> 点击“确定”，然后再双击运行。
3. **直接跳过拦截**：如果在运行 `.bat` 时弹出了蓝色的拦截框，你可以直接点击文字 **“更多信息”**，然后下方会出现一个 **“仍要运行”** 的按钮，点击即可。
